## 1. Описание деплоя

### Какой способ выбрала

Подготовлены **два варианта** деплоя:

1. **Docker + docker-compose** — основной способ. Одна команда — и бот работает на любом сервере с автоперезапуском.
2. **Railway** — облачный вариант. Бесплатный тариф, не нужен свой сервер.

### Почему Docker

- Воспроизводимость: одинаковое окружение на любой машине
- `docker-compose up -d` — запуск одной командой
- `restart: unless-stopped` — бот автоматически перезапустится после сбоя
- Легко переносить между серверами

## 2. Процесс деплоя

### Пошаговая инструкция (Docker)

1. Создан `Dockerfile`:
   ```dockerfile
   FROM python:3.11-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   CMD ["python", "bot.py"]
   ```

2. Создан `docker-compose.yml`:
   ```yaml
   version: "3.8"
   services:
     bot:
       build: .
       env_file:
         - .env
       restart: unless-stopped
   ```

3. Создан `.env` с `BOT_TOKEN` и `TMDB_API_KEY`.

4. Запуск:
   ```bash
   docker-compose up -d
   ```

5. Проверка:
   ```bash
   docker-compose logs -f bot
   ```

### С какими проблемами столкнулись

| Проблема | Решение |
|---|---|
| Контейнер перезапускался — невалидный TMDB API key | Перегенерировала ключ на themoviedb.org |
| `tracker.db` пересоздавалась при каждом ребилде образа | Данные хранятся в контейнере; для прода нужен volume |
| Логи не показывали ошибки | `logging.basicConfig` выводит в stdout — Docker подхватывает |

## 5. Улучшения на основе фидбека

### Что изменили

**1. Inline-кнопки ➕ для добавления в watchlist (по фидбеку Пользователя 2)**

Самое важное улучшение. Под каждым списком результатов теперь появляются кнопки `[➕ 1] [➕ 2] ... [➕ 10]`. При нажатии фильм добавляется в watchlist без ввода команд. Callback data: `add_{type}_{tmdb_id}`.

```python
for i, item in enumerate(items, 1):
    buttons.append(
        InlineKeyboardButton(f"➕ {i}", callback_data=f"add_{item['type']}_{item['id']}")
    )
```

После нажатия — всплывающее уведомление `«Интерстеллар» добавлен!` без перезагрузки страницы.

**2. Кнопочное меню в /start (по фидбеку Пользователя 4)**

Добавлена inline-клавиатура `MAIN_MENU` с кнопками: Поиск, Популярное, Тренды, Лучшее, Буду смотреть, Просмотрено, Помощь. При нажатии — контент показывается прямо в сообщении через `edit_message_text`.

```python
MAIN_MENU = InlineKeyboardMarkup([
    [InlineKeyboardButton("🔍 Поиск", callback_data="m_search"),
     InlineKeyboardButton("🔥 Популярное", callback_data="m_popular")],
    [InlineKeyboardButton("📈 Тренды", callback_data="m_trending"),
     InlineKeyboardButton("🏆 Лучшее", callback_data="m_top")],
    [InlineKeyboardButton("📋 Буду смотреть", callback_data="m_wl"),
     InlineKeyboardButton("⭐ Просмотрено", callback_data="m_watched")],
    [InlineKeyboardButton("📖 Помощь", callback_data="m_help")],
])
```

**3. Описание фильма в результатах (по фидбеку Пользователя 1)**

К каждому результату поиска добавлен `overview` — краткое описание до 150 символов курсивом:

```python
def fmt_item(item, idx):
    line = f"{idx}. 🎬 *{item['title']}* ({item['year']}) ⭐ {item['rating']:.1f}"
    if item.get("overview"):
        line += f"\n    _{item['overview']}_"
    return line
```

**4. Рейтинг TMDB в watchlist (по фидбеку Пользователя 3)**

При добавлении в watchlist сохраняется `tmdb_rating`. В `/watchlist` рейтинг отображается рядом с названием:

```
#1 🎬 Интерстеллар (2014) ⭐ 8.7
#2 📺 Во все тяжкие (2008) ⭐ 8.9
```

**5. Расширенная статистика (по фидбеку Пользователя 5)**

Команда `/stats` теперь показывает полную картину: количество пользователей, общее число команд, размер watchlist, количество оценённых фильмов и отзывов. Данные берутся из таблицы `usage_log`.
